# Resume Template Notes

## From `이력서 양식(엔지니어)_송부.docx`

- 전형적인 제출용 이력서 구조를 요구함
- 섹션 순서가 명확함: 개인정보 -> 학력 -> 경력 -> 핵심역량 -> 기술스택 -> 경력 세부 -> 기타 -> 자기소개
- 경력 세부는 아래 블록 구조가 잘 맞음
  - 회사 소개
  - 담당 업무
  - 주요 업무/성과
  - 이직사유

## Practical implications

- 현재 웹 이력서를 제출용 이력서로 변환할 때도 "핵심역량"과 "경력 세부"는 분리하는 편이 좋음
- 핵심역량은 4~6개 bullet로 압축 가능해야 함
- 경력 세부는 각 프로젝트를 3~5 bullet 수준으로 유지해야 읽힘
- 제출용 문서에서는 기술 약어보다 결과/책임/영향 중심 문장이 더 잘 맞음

## From `이력서_정주안.docx`

- 현재 제출용 초안은 이미 아래 방향을 반영하고 있음
  - 글로벌 런칭
  - 운영 안정성
  - 자동화와 비용 절감
  - 배포/테스트 자동화
- 다만 일부 표현은 여전히 내부 기술 용어 비중이 높음
  - Kafka partition key
  - Provider 추상화
  - stateless worker
  - Custom Metric 기반 오토스케일링

## Recommended use

- 웹 이력서 문구를 다듬을 때는 `이력서_정주안.docx`의 섹션 구조를 참고하되
- 기술 서술은 현재처럼 더 recruiter-friendly 하게 번역해 쓰는 편이 좋음
